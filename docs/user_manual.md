# HomeCam 使用说明

## 1. 简介

**HomeCam** 是一款将 Android 手机变身为智能网络摄像头的应用。它利用手机的摄像头进行实时监控，内置多种 AI 检测功能（人物移动、婴儿哭声、睡眠、跌倒等），支持 Web 浏览器和 RTSP 协议远程查看。

**HomeCam-TE** 是配套的 Android 显示终端，用于在局域网内同时监控多台 HomeCam 设备，支持语音+振动报警、摄像头控制、历史录像查看等功能。

> **核心特点**：完全免费 · 无广告 · 本地 AI 处理（数据不上传云端） · 开源 · Server + Terminal 双端协作

---

## 2. 安装与启动

### 2.1 安装
- 下载 APK 安装包，在 Android 手机上安装
- 支持 Android 8.0（API 26）及以上版本

### 2.2 启动
1. 打开应用，点击 **"启动服务"**
2. 应用将启动摄像头预览，并在后台运行 Web 服务器和 RTSP 服务器
3. 启动后，手机屏幕可关闭（应用持有关键性唤醒锁）

### 2.3 连接方式

**局域网内访问**（推荐）：
- 打开手机热点或将手机和电脑连接到同一 Wi-Fi
- 浏览器访问 `http://手机IP:8080`（手机 IP 可在应用首页上方查看）
- 或使用 RTSP 播放器连接 `rtsp://手机IP:8554/live`

**远程访问**：
- 需在路由器上设置端口转发（转发 8080 和 8554 端口）
- 或使用内网穿透工具（如 frp、Tailscale）

## 3. 实时监控

### 3.1 手机端界面
- **摄像头预览**：启动后自动显示实时画面
- **切换摄像头**：点击 "切换摄像头" 弹出列表，可切换前后摄像头
- **录像开关**：控制是否录制事件视频
- **帮助按钮**：右上角 `ⓘ` 打开关于/捐赠/开源信息对话框

### 3.2 Web 端界面

浏览器打开后有两个选项卡：

**实时（Live）选项卡**：
- **实时画面**：自动加载 MJPEG 视频流
- **摄像头选择**：下拉菜单切换前后摄像头
- **电源开关**：`⚡` 开启/关闭摄像头电源（关闭后省电）
- **旋转按钮**：`↻` 每次点击画面顺时针旋转 90°
- **电量显示**：右上角显示手机当前剩余电量
- **事件日志**：显示最近的 AI 检测事件

**历史（History）选项卡**：
- **视频列表**：显示所有录制的视频，带缩略图、时间、事件类型、时长、大小
- **视频播放**：点击列表项播放录制视频
- **下载**：点击 `⬇` 下载视频文件

## 4. HomeCam-TE 显示终端

**HomeCam-TE** 是专为 HomeCam 设计的 Android 显示终端，支持在一台设备上同时监控多台 HomeCam 摄像头。采用 Jetpack Compose + Material3 构建，提供卡片式网格监控、语音+振动报警、历史录像回放等功能。

### 4.1 快速开始

| 步骤 | 操作 |
|------|------|
| 1 | 在 Android 设备上安装 HomeCam-TE APK |
| 2 | 确保 TE 设备和 HomeCam 设备在同一局域网 |
| 3 | 点击右上角 `+` 手动添加摄像头（IP + 端口），或点击搜索图标自动发现 |
| 4 | 连接成功自动显示视频画面 |

### 4.2 多设备监控网格

| 设备数量 | 布局 | 说明 |
|---------|------|------|
| 1 台 | 单列 | 卡片占满宽度 |
| 2 台及以上 | 双列 | 始终双列，奇数台时最后一张在左下角 |

**卡片结构**：

```
┌──────────────────────────────┐
│ [设备名称/IP]       [⚡电源]  │
│                              │
│    MJPEG / RTSP 视频画面     │
│    (9:16 竖屏, FIT_CENTER)   │
│                              │
│                    [🔄镜头]   │
├──────────────────────────────┤
│ 最新事件文本     [📁] [＞]    │
└──────────────────────────────┘
```

- 视频画面支持 MJPEG 和 RTSP 两种协议，长按菜单可切换
- 点击画面进入全屏预览，长按弹出菜单（编辑设备/切换视频流格式/删除设备）
- 点击文件夹图标进入历史录像查看
- 点击展开箭头查看最近 20 条事件详情

### 4.3 设备发现与添加

**手动添加**：点击右上角 `+`，输入 IP 地址和端口（默认 8080），可选设备名称。

**UDP 自动发现**：点击搜索图标，TE 端向局域网广播 `HOMECAM_DISCOVER` 消息，HomeCam 设备收到后自动回复。3 秒收集窗口，按设备 ID 去重，重复发现不会重复添加。

**设备持久化**：添加的设备自动保存到 Room 数据库，重启应用自动恢复连接。

### 4.4 事件监控与报警

**事件同步**：每 5 秒轮询 `/api/events`，增量更新（时间戳比较），内存保留最近 20 条。

**语音报警（TTS）**：使用 Android TTS API，中文播报事件类型。

**振动报警**：不同事件不同振动时长。

| 事件类型 | TTS 播报内容 | 振动时长 |
|---------|-------------|---------|
| 有人进入 | "有xxx进入了" | 200ms |
| 有人离开 | "有xxx离开了" | 200ms |
| 婴儿哭声 | "检测到婴儿哭声" | 1000ms |
| 宝宝睡着 | "宝宝睡着了" | 100ms |
| 宝宝睡醒 | "宝宝睡醒了" | 100ms |
| 检测到摔倒 | "检测到有人摔倒" | 1000ms |
| 有人站起 | "有人站起来了" | 100ms |
| 玩手机 | "有人在玩手机（XX%）" | 200ms |

**报警开关**：设置页面支持总开关、按事件类型独立开关、语音/振动独立开关。

### 4.5 摄像头控制

| 操作 | 功能 |
|------|------|
| 点击电源按钮 | 开启/关闭摄像头（关闭后 MJPEG 流自动刷新） |
| 点击镜头切换 | 切换前后/广角摄像头（仅多镜头设备可见） |
| 长按菜单 → 视频流格式 | 切换 MJPG / RTSP 协议，按设备记忆选择 |
| 长按菜单 → 编辑设备 | 修改名称、IP、端口（改 IP/端口自动重连） |

### 4.6 历史录像查看

点击卡片下方的文件夹图标进入历史录像页面：

- **上半部分**：录像列表，显示缩略图（`/api/thumbnails`）、事件类型彩色标签、时间、时长、文件大小
- **下半部分**：ExoPlayer MP4 播放器，支持播放控制和进度拖动
- 点击列表项自动切换播放内容

### 4.7 设置与配置

| 设置项 | 说明 |
|--------|------|
| 报警总开关 | 开启/关闭所有报警 |
| 按事件类型开关 | 分别控制 6 类事件的报警 |
| 振动开关 | 独立控制振动 |
| 语音开关 | 独立控制 TTS 语音播报 |

所有 AI 检测均在手机本地运行，**无需联网，不上传数据**。

### 4.1 人物移动检测（默认开启）
- 通过 MediaPipe Object Detection 模型检测人/猫/狗/鸟等目标
- 检测到目标进入 → 记录 `"有人进入了"` 事件
- 目标离开超过 2 分钟 → 记录 `"有人离开了"` 事件
- **设置项**：`detection_target` 可选择检测目标（人/猫/狗/鸟）

### 4.2 婴儿哭声检测（默认关闭）
- 通过 YAMNet 音频分类模型实时分析麦克风声音
- 检测到婴儿哭声 → 记录 `"婴儿哭声"` 事件
- 注意：需要手机麦克风权限

### 4.3 睡眠检测（默认关闭）
- **双通道检测**：
  - **面部通道**：通过 Face Landmarker 分析眼部纵横比（EAR）。闭眼 > 5 帧 → `"宝宝睡着了"`；睁眼 → `"宝宝睡醒了"`
  - **姿态辅助通道**：面部不可见时（宝宝侧睡/背对镜头），通过躯干角度判断。躯干水平持续 30 秒 → 自动判定为睡眠；坐起（躯干直立）→ 判定为睡醒
- 睡眠状态下，跌倒检测会自动暂停，避免躺姿误报

### 4.4 跌倒检测（默认关闭）
- 通过 Pose Landmarker 分析人体躯干角度
- 躯干水平超过阈值 → `"检测到有人摔倒"`；重新站直 → `"有人站起来了"`
- **手机安装方向设置**：如果手机横置安装（如横放在墙上），需在设置中将 `landscape_mode` 改为 "横屏"，否则跌倒检测会误报

### 4.5 手机使用检测（默认关闭）
- 通过 Object Detection（手机检测）+ Hand Landmarker（手部接近度）联合判断
- 检测到有人手持手机 → `"有人在玩手机（XX%）"`
- 冷却时间 30 秒，避免频繁触发

### 4.6 检测性能设置
- **推理后端**：可选择 CPU 或 GPU（GPU 更快但更耗电）
- **检测间隔**：每 N 帧检测一次，值越大越省电（1-10 可选，默认 3）
- **检测叠加层**：开启后在视频流上显示检测框和标签

## 5. 录像与回放

### 5.1 录像策略

| 策略 | 触发方式 |
|------|---------|
| 事件触发（默认） | 仅在检测到特定事件时录制（哭声、睡眠、跌倒、手机使用、进入/离开） |
| 有人/动物存在 | 检测到任何移动目标时持续录制，但不在日志中记录 "移动" 事件 |

### 5.2 录像设置
- **保存时长**：事件触发前后各录制 N 秒（2-5 秒可选，默认 3 秒）
- **最大视频数量**：超出数量自动删除最旧视频（10-1000 可选，默认 50）

### 5.3 查看录像
- **手机端**：点击应用主界面 "录像" 按钮打开图库
- **Web 端**：切换到 "历史" 选项卡浏览、播放、下载
- **HomeCam-TE**：点击卡片文件夹图标进入历史录像页面

## 6. 网络串流

### 6.1 MJPEG 流（Web 浏览器）
- 默认开启，端口 8080
- 通过 `http://手机IP:8080` 在任意浏览器中查看
- 自适应 JPEG 质量（根据帧间隔自动调整 45-75）

### 6.2 RTSP 流（专业播放器）
- 默认关闭，需在设置中启用
- 端口 8554
- 支持工具：VLC、ffplay、HomeCam-TE 等
- URL：`rtsp://手机IP:8554/live`
- 支持 RTP/UDP 和 RTP/TCP 传输
- H.264 编码，1Mbps 码率

### 6.3 局域网自动发现
- 应用在 UDP 端口 45678 监听 `HOMECAM_DISCOVER` 广播
- 收到后回复设备名称、IP、端口等信息
- 配合 HomeCam-TE、Home Assistant 等工具可实现自动发现

## 7. 设置说明

### 7.1 视频设置
| 设置项 | 选项 | 说明 |
|--------|------|------|
| 分辨率缩放 | 1.0 / 0.75 / 0.5 | 降低分辨率可提升性能 |
| 帧率 | 15 / 30 FPS | 15 FPS 更省电 |

### 7.2 网络设置
| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| Web 端口 | 8080 | HTTP/MJPEG 端口 |
| RTSP 端口 | 8554 | RTSP 直播端口 |
| RTSP 开关 | 关闭 | 启用/禁用 RTSP |
| MJPEG 开关 | 开启 | 启用/禁用 Web 视频流 |

### 7.3 检测设置
| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 人物移动 | 开启 | 主要检测开关 |
| 检测目标 | person | person/cat/dog/bird |
| 婴儿哭声 | 关闭 | 需要麦克风权限 |
| 睡眠检测 | 关闭 | 面部+姿态双通道 |
| 跌倒检测 | 关闭 | 注意手机方向设置 |
| 手机使用 | 关闭 | 检测手持手机 |
| 推理后端 | CPU | CPU 更稳定，GPU 更快 |
| 检测间隔 | 3 帧 | 越大越省电 |

### 7.4 录像设置
| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 录像策略 | 事件触发 | 见第 5 节 |
| 保存时长 | 3 秒 | 事件前后各 N 秒 |
| 最大视频数 | 50 | 自动清理旧视频 |

### 7.5 其他设置
| 设置项 | 默认值 | 说明 |
|--------|--------|------|
| 检测叠加层 | 开启 | 视频流上显示检测框 |
| 手机方向 | 竖屏 | 跌倒检测需要此设置 |

## 8. Web API 附录

HomeCam 内置 HTTP 服务器提供以下 REST API：

| 接口 | 说明 |
|------|------|
| `GET /api/status` | 服务状态（运行状态、IP、端口、检测模式、电量等） |
| `GET /api/events` | 事件历史列表 |
| `GET /api/cameras` | 可用摄像头列表 |
| `GET /api/camera/switch?cameraId=X` | 切换摄像头 |
| `GET /api/camera/power?action=on\|off` | 电源控制 |
| `GET /api/videos` | 视频文件列表 |
| `GET /video` | MJPEG 视频流 |
| `GET /api/frame.jpg` | 当前帧 JPEG 快照 |
| `GET /api/thumbnails/{filename}` | 视频缩略图 |
| `GET /videos/{filename}` | 视频文件下载 |

## 9. 常见问题

**Q：画面卡顿/延迟大？**
- 降低分辨率缩放（0.75 或 0.5）
- 降低帧率到 15 FPS
- 关闭不必要的 AI 检测功能
- 使用 5G Wi-Fi 而非 2.4G

**Q：跌倒检测一直误报？**
- 检查手机安装方向：如果手机横着放，需在设置中将 `landscape_mode` 改为 "横屏"
- 睡眠检测开启时，跌倒检测会自动在睡眠状态下暂停

**Q：RTSP 连接不上？**
- 确认已在设置中开启 RTSP 开关
- 确认防火墙未阻止 8554 端口
- 部分播放器需尝试切换 RTP/UDP 和 RTP/TCP

**Q：HomeCam-TE 搜索不到设备？**
- 确认 TE 和 HomeCam 在同一局域网
- 确认 HomeCam 服务已启动
- 部分路由器隔离了广播包，可尝试手动添加 IP

**Q：应用被杀后台？**
- 在系统设置中将 HomeCam 加入省电白名单
- 锁定应用在多任务列表中
- 部分厂商（小米、华为、OPPO 等）需在系统管家设置中允许后台运行

**Q：录像文件保存在哪里？**
- HomeCam 端：`Android/data/com.homecam.app/files/HomeCam/`
